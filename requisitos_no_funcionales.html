REQUISITOS NO FUNCIONALES

Categoría RNF (Lenguaje Técnico) Justificación (Necesidad Implícita del HU) Historias de Usuario Implícitas
Seguridad Confidencialidad de Datos. Los datos de registro deben ser cifrados en reposo (at rest) para proteger la identidad del usuario. La confianza empieza con la protección de la información inicial. HU01
Datos Integridad Referencial. Debe implementarse un mecanismo de validación estricta para asegurar que los campos esenciales del perfil estén completos. El perfil debe ser fiable para poder ser "mostrado" o usado por el sistema. HU02
Seguridad Robustez de Autenticación. Implementación de mecanismos de mitigación contra ataques de fuerza bruta y rate limiting en el endpoint de inicio de sesión. Asegura que el acceso al sistema sea fiable y seguro. HU03
Rendimiento Disponibilidad de Servicios de Reporte. El servicio de procesamiento de datos para reportes debe tener una disponibilidad mínima del 99.9% (tres nueves). La organización necesita absoluta fiabilidad para obtener sus métricas en cualquier momento. HU04
Rendimiento Latencia de Búsqueda Geográfica. El tiempo de respuesta para las consultas de búsqueda basada en geolocalización no debe superar los 1.5 segundos. La inmediatez en los resultados es clave para la usabilidad. HU05
Usabilidad Precisión Semántica del Filtrado. El sistema debe asegurar que los resultados de filtrado coincidan exactamente con la categoría de interés seleccionada. La calidad del resultado es vital para la motivación del usuario. HU06
Escalabilidad Capacidad del Algoritmo. El motor de recomendación debe ser capaz de procesar hasta 10,000 perfiles de usuario en paralelo manteniendo la calidad del matching. Preparación para el crecimiento masivo del sistema. HU07
Usabilidad Consistencia de Interfaz de Publicación. El flujo de creación de actividades debe ser uniforme y guiado, minimizando errores por parte de la Organización. Facilita la adopción de la plataforma por parte de las organizaciones. HU08
Seguridad Control de Acceso (RBAC). El sistema debe aplicar reglas de autorización estricta para que una Organización solo pueda modificar el contenido que le pertenece. Asegura la integridad de los datos publicados. HU09
Usabilidad Mínima Fricción de Interacción. El diseño de la plataforma debe priorizar el número mínimo de clics para realizar tareas clave (menor a 4). Respuesta directa a la HU10: Facilidad de uso para usuarios menos familiarizados con la tecnología. HU10
Rendimiento Tasa de Entrega de Notificaciones. La entrega de correos de confirmación debe tener una tasa de éxito del 99% y una latencia de menos de 30 segundos. La confirmación debe ser fiable e inmediata para el usuario. HU11
Accesibilidad Diseño Gráfico de Historial. La visualización del historial debe ser clara y contrastada para facilitar la lectura por parte de usuarios con dificultades visuales. El historial debe ser comprensible y legible para que pueda ser "mostrado". HU12
Rendimiento Precisión de Recordatorios. El servicio de alertas debe garantizar que los recordatorios por correo/notificación se envíen con una desviación máxima de 5 minutos respecto a la hora programada. La puntualidad es crítica para la asistencia a eventos presenciales. HU13
Rendimiento Rendimiento de Carga de Archivos. El tiempo de subida de imágenes/videos (hasta 5MB) debe ser menor a 5 segundos para evitar el abandono del flujo. La subida rápida es clave para que el usuario complete su reporte. HU14
Mantenibilidad Modularidad de Módulos Nuevos. El módulo de micro-reportes debe ser autocontenido y desacoplado para facilitar futuras actualizaciones o reemplazos. Simplifica el mantenimiento del código de esta nueva funcionalidad. HU15
