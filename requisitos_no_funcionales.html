REQUISITOS NO FUNCIONALES

Categoría RNF (Lenguaje Técnico) Justificación (Necesidad Implícita del HU) HU Relacionada (Corregida)
Rendimiento Tiempo de Respuesta de Búsqueda: Las búsquedas con filtros deben mostrar resultados en menos de 5 segundos con hasta 500 actividades, usando consultas SQL con índices. La rapidez en encontrar oportunidades relevantes evita que el voluntario abandone la búsqueda. HU01
Usabilidad Validación Simple de Formularios: El formulario de creación de actividades debe usar validación HTML5 y mostrar mensajes claros cuando falten datos. Los mensajes claros evitan que las organizaciones publiquen actividades con información incompleta. HU02
Rendimiento Respuesta Rápida de Inscripción: El botón de inscripción debe ejecutar la operación y mostrar confirmación en menos de 3 segundos. La respuesta rápida mantiene el interés del voluntario en el momento crítico de compromiso. HU03
Precisión Exactitud de Cálculos Básicos: Las consultas SQL de reportes (SUM, COUNT) deben calcular correctamente los totales sin redondeos. Los números correctos son importantes para que las organizaciones confíen en los datos mostrados. HU04
Usabilidad Diseño Simple de Lista: La tabla HTML de eventos debe tener encabezados claros, usar CSS básico para alternar colores de filas, y ordenar por fecha de más próxima a más lejana. Una lista clara y ordenada ayuda al voluntario a identificar rápidamente sus próximos compromisos. HU05
Rendimiento Actualización Simple de Estado: Cuando se aprueba o rechaza una solicitud, el cambio en la base de datos (UPDATE) debe reflejarse inmediatamente al recargar la página. El cambio rápido de estado permite a la organización gestionar solicitudes eficientemente. HU06
Rendimiento Envío de Emails Funcional: Los emails deben enviarse exitosamente usando la configuración SMTP básica. Las notificaciones por email mantienen informado al voluntario sin necesidad de entrar constantemente a la plataforma. HU07
Escalabilidad Envío de Múltiples Emails Secuencial: El sistema debe poder enviar emails a hasta 50 voluntarios en un loop simple. Para un proyecto universitario, poder enviar mensajes a grupos pequeños-medianos de voluntarios es suficiente. HU08
Seguridad Validación Básica de Comentarios: El formulario de reseñas debe validar que el campo de texto no esté vacío y limitar la longitud a 500 caracteres. Para la imagen, validar que sea formato jpg/png y no supere 2MB. Validaciones simples previenen errores comunes y contenido problemático. HU09
Usabilidad Diseño Simple y Limpio: La interfaz debe usar CSS básico con fuentes legibles, colores con buen contraste, botones claramente visibles y formularios bien espaciados. Un diseño sencillo y claro facilita el uso para personas con diferentes niveles de habilidad tecnológica. HU10
Precisión Coincidencia Simple de Intereses: El sistema debe enviar emails solo a voluntarios cuyo campo de intereses coincida exactamente con la categoría de la nueva actividad, usando un WHERE simple en SQL. Enviar notificaciones relevantes aumenta la probabilidad de que los voluntarios se interesen en participar. HU11
Rendimiento Carga Rápida de Estadísticas: La página de estadísticas debe cargar en menos de 5 segundos ejecutando máximo 5 consultas SQL simples y precalculadas. Las estadísticas rápidas permiten a la organización tomar decisiones sin frustrarse esperando que cargue. HU12
Datos Integridad Básica del Historial: El historial debe mostrar solo actividades con estado='completado', y no se debe permitir que el voluntario edite o elimine registros. Un historial confiable y no modificable le da valor como registro verificable de experiencia. HU13
Seguridad Efecto Inmediato de Pausa: Cuando el campo "activo" está en false, todas las páginas que muestren actividades deben incluir un condicional IF que filtre las actividades de organizaciones pausadas. La funcionalidad de pausa debe funcionar correctamente para evitar confusiones y solicitudes no deseadas. HU14
Seguridad Privacidad Básica de Mensajes: Los mensajes deben estar vinculados en la BD con claves foráneas y solo mostrarse a los participantes de la conversación, usando sesiones o tokens simples. La privacidad básica genera confianza para compartir información logística. HU15
